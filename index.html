Overview


The main purpose of this project was an application for data visualisation, but we branched out into something more commercial - something more useful, like budgeting. This project can also be used for sole visualization purposes, though. What's unique about Project PERFOS?

1. Pure Python
2. Simplicity
3. Visual feedback

Pure Python


The application uses ONLY python - since the purpose is data analysis, and the go-to in our opinion was Python, we thought, "Why not make the whole thing using just Python?" We haven't used many libraries, however - Only SQLite3, Tkinter, Pandas and Matplotlib. Which leads us to the next feature - Simplicity.

Simplicity


The code of this application is less than a total of 400 lines and uses a unique interface to simplify usage as well as processes.


  • Graphs are displayed in messageboxes(Tkinter) upon respective button press.
  • Transaction history is displayed in a disabled text area to the right which is updated in real time, and the data is fetched from the database and stored in a pandas dataframe (which is the content of the text area).
  • The database, made using SQLite3, is stored locally and can be cleared using the 'Clear History' button.
  • Income/Expense as well as Payment records are input through the GUI which are then stored in the database.
  • Goal tracking: When the user's balance reaches the set goal amount, the application resets the goal value and displays the respective success message.

Visual analytics


The data within the database is analyzed and visualised (which is Python's cup of tea) and three types of graphs are made which can be accessed using the respective buttons.

a. Line graph - Income vs Expenses
b. Pie chart - Expense category distribution
c. Bar Graph - Savings against Income

Limitations


1. Data Security: Storing sensitive financial data requires encryption and security measures, however since the database is stored locally the security of the entered data will depend on the user's system's own security.
2. Handling Large Data: If a user has a lot of data, efficient data management and visualization can become complex and could take a while (Python is pretty slow).

Quick start guide


Click 'Load/Refresh' in the transactions history section to display past entries. The database will initially contain 4-5 sample entries. Click the Clear button to clear them and then make your own entries. They will be saved and available the next time you open the application.


Click any of the buttons in the 'Visualisation' area to create the respective graph from the data within the database. It will be displayed in a messagebox.
